Classification

We're solving a classification problem

is this iris versicolor, setosa or virginica?

There are other kinds...

  • regression, like predicting a numeric house price
  • unsupervised, when we don't know the answers

the iris problem is:

  • classification (what species is this one?)
  • supervised (we have a dataset with correct answers in it)

Examples

To learn a supervised problem, we need examples

5.1,3.5,1.4,0.2,Iris-setosa

  • Four Features:
    • Petal width & length
    • Sepal width & length
  • One Label (Setosa | Versicolor | Virginica)
  • examples consist of four floating point numbers, plus a text label
  • there are 150 examples in the data set

Training and Testing

Training

  • training examples are fed to the ML algorithm
  • a model is built that can predict for new examples

Testing

  • the model predicts for examples it hasn't seen before
  • "goodness" of the model is assessed
  • goodness is a more complex problem than you'd think

The kNN classifier

We're going to use a classic algorithm:

a k-Nearest Neighbours classifier.

It looks at how similar a new example is to the examples it trained with, and classifies as the same as the most-similar 'k' training examples.

The kNN classifier

  • Each training example is plotted in a different colour for each species.
  • Four of the five nearest data points are blue, so the new example is classified as blue.

The kNN classifier

  • Each training example is plotted in a different colour for each species.
  • Predictions are made over the whole x-y space and regions coloured in a lighter version of the training example colours.
  • The clear, striking stripes show that these features are pretty good at differentiating the species.

Training

Most algorithms have a 'training' phase where they deduce and optimise a target function.

The k-NN classifier doesn't really have a training phase as it just 'memorizes' the training data.

Validation

...but is it good at predicting the species?

How do we measure the effectiveness of our algorithm?

Train/Test Split

  • assign some examples to a "training" set (say 70%)
  • and the rest to a "test" set (say 30%)
  • have the algorithm memorize the training set
  • predict the classes of the test set
  • how many did it get right? (%)
  • simple approach
    • you need to assign examples to the train/test sets randomly!

k-Fold Cross-Validation

Make better use of your data!

  • choose say k = 5, then
  • randomly assign examples to 5 equal sized "folds"
  • train with 4/5 "folds", test with the other
  • 5 times
  • average the % correct
  • reuse your valuable data set k times!
